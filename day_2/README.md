# День 2: Эмбеддинги и скрытые состояния (Hidden States)

## 1. Что такое эмбеддинги?

**Эмбеддинги (Embeddings)** — это векторные представления слов (или токенов) в многомерном пространстве. В отличие от простых ID токенов, эмбеддинги отражают семантический смысл: слова с похожим значением в этом пространстве будут находиться "близко" друг к другу.

В трансформерах есть несколько уровней представлений:
- **Token Embeddings:** Начальные векторы токенов из словаря.
- **Hidden States (Скрытые состояния):** Векторы на выходе каждого слоя трансформера. Они учитывают контекст (благодаря механизму Attention).
- **Pooler Output:** Агрегированное представление всего предложения (обычно берется вектор токена `[CLS]` и прогоняется через дополнительный линейный слой).

## 2. Извлечение Hidden States

Современные модели (например, BERT) состоят из множества слоев (обычно 12 для `base` версий). Каждый слой уточняет представление токена.

Чтобы получить скрытые состояния всех слоев в библиотеке `transformers`, при загрузке модели или при вызове нужно указать параметр `output_hidden_states=True`.

## 3. Близость векторов (Cosine Similarity)

Для сравнения эмбеддингов часто используют **косинусное сходство**. Оно показывает, насколько векторы сонаправлены, независимо от их длины. Значение варьируется от -1 до 1 (где 1 — полная идентичность).
